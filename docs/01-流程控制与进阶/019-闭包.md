# 闭包

闭包（Closure）是一种匿名函数，它可以赋值给变量也可以作为参数传递给其它函数。不同于函数的是，它允许捕获调用者作用域中的值。例如：

```Rust
fn main() {
   let x = 1;
   let sum = |y| x + y;
   assert_eq!(3, sum(2));
}
```

上面的代码展示了闭包的简单使用方式：使用 `||` 代替 `()` 将参数包裹起来，其余表达式跟函数定义一致。闭包可以捕获外部变量 `x`，这在普通函数中是不允许的。

# 闭包语法

闭包虽然本质上是一个匿名函数，但是并不会像函数那样在声明时规定参数和返回值的类型，因为闭包通常很短小，而且是在具体的上下文中使用，所以编译器完全有能力推断出参数和返回值的类型。

```Rust
fn  add_one_v1   (x: u32) -> u32 { x + 1 }
let add_one_v2 = |x: u32| -> u32 { x + 1 };
let add_one_v3 = |x|             { x + 1 };
let add_one_v4 = |x|               x + 1  ;
```

以上展示了四种不同的写法，效果相同，对应的类型推断也依次增多。第一行是标准的函数定义，后面三行是不同形式的闭包定义。可以观察到：闭包的参数和返回值类型都可以省略，甚至当闭包体只有一个表达式时，`{}` 也可以省略。

不过，有时候编译器也无法推断类型：

```Rust
let example_closure = |x| x;

let s = example_closure(String::from("hello"));
let n = example_closure(5);
```

第一次调用时编译器推断 `x` 的类型为 `String`，第二次调用传入 `5` 的时候就会报错，因为编译器已经将闭包的参数类型绑定为 `String` 了：

```Plain
error[E0308]: mismatched types
 --> src/main.rs:5:29
  |
5 |     let n = example_closure(5);
  |                             ^
  |                             |
  |                             expected struct `String`, found integer
```

所以**在编译器无法推断类型时，我们需要手动给出类型标注**：

```Rust
let example_closure = |x: i32| -> i32 { x };
```

## 结构体中的闭包

假设我们要实现一个简单的缓存功能：只关心最昂贵（最耗时）的计算结果，其它的结果直接忽略。

先看一个使用函数实现的版本：

```Rust
fn generate_workout(intensity: u32, random_number: u32) {
    if intensity < 25 {
        println!("Today, do {} pushups!", simulated_expensive_calculation(intensity));
        println!("Next, do {} situps!", simulated_expensive_calculation(intensity));
    } else {
        if random_number == 3 {
            println!("Take a break today! Remember to stay hydrated!");
        } else {
            println!(
                "Today, run for {} minutes!",
                simulated_expensive_calculation(intensity)
            );
        }
    }
}

fn simulated_expensive_calculation(intensity: u32) -> u32 {
    println!("calculating slowly...");
    thread::sleep(Duration::from_secs(2));
    intensity
}
```

上述代码中，当 `intensity < 25` 时，`simulated_expensive_calculation` 被调用了两次，而这是一个很耗时的操作。如果改用闭包，我们可以做到惰性求值（只在需要时才计算）：

```Rust
fn generate_workout(intensity: u32, random_number: u32) {
    let expensive_closure = |num| {
        println!("calculating slowly...");
        thread::sleep(Duration::from_secs(2));
        num
    };

    if intensity < 25 {
        println!("Today, do {} pushups!", expensive_closure(intensity));
        println!("Next, do {} situps!", expensive_closure(intensity));
    } else {
        if random_number == 3 {
            println!("Take a break today! Remember to stay hydrated!");
        } else {
            println!(
                "Today, run for {} minutes!",
                expensive_closure(intensity)
            );
        }
    }
}
```

但这样依然存在问题：当 `intensity < 25` 时闭包仍然被调用了两次。这时可以借助结构体来缓存闭包的结果：

```Rust
use std::thread;
use std::time::Duration;

struct Cacher<T>
where
    T: Fn(u32) -> u32,
{
    calculation: T,
    value: Option<u32>,
}

impl<T> Cacher<T>
where
    T: Fn(u32) -> u32,
{
    fn new(calculation: T) -> Cacher<T> {
        Cacher {
            calculation,
            value: None,
        }
    }

    fn value(&mut self, arg: u32) -> u32 {
        match self.value {
            Some(v) => v,
            None => {
                let v = (self.calculation)(arg);
                self.value = Some(v);
                v
            }
        }
    }
}
```

通过 `Cacher` 结构体，闭包只会在第一次调用时执行，之后直接返回缓存的结果。

# 捕获变量

闭包可以从其所在的环境中捕获变量，这是闭包与普通函数最大的不同。捕获有三种方式，恰好对应三种借用方式：

| 捕获方式 | 对应借用 | 说明 |
|---------|---------|------|
| 不可变引用 | `&T` | 闭包读取变量但不修改 |
| 可变引用 | `&mut T` | 闭包修改变量 |
| 获取所有权 | `T` | 闭包接管变量所有权 |

## 不可变引用捕获

```Rust
fn main() {
    let x = 4;
    let equal_to_x = |z| z == x;

    let y = 4;
    assert!(equal_to_x(y));
}
```

闭包 `equal_to_x` 仅仅读取了 `x` 的值，因此以不可变引用的方式捕获 `x`。

## 可变引用捕获

```Rust
fn main() {
    let mut list = vec![1, 2, 3];
    println!("Before defining closure: {:?}", list);

    let mut borrows_mutably = || list.push(7);

    borrows_mutably();
    println!("After calling closure: {:?}", list);
}
```

闭包 `borrows_mutably` 修改了 `list`，因此以可变引用的方式捕获它。注意在调用闭包之前，不能有其它对 `list` 的不可变引用存在（比如上一行的 `println!` 必须在闭包定义前结束使用）。

## 获取所有权捕获

```Rust
fn main() {
    let x = vec![1, 2, 3];
    let equal_to_x = move |z| z == x;

    // println!("can't use x here: {:?}", x); // 会报错，x 的所有权已被转移
    let y = vec![1, 2, 3];
    assert!(equal_to_x(y));
}
```

在闭包参数列表前加上 `move` 关键字，可以强制闭包获取它所使用的环境中变量的所有权。这在将闭包传递到新线程中时非常有用，因为新线程可能在原线程结束后才执行，所以需要获取所有权来确保数据的有效性。

编译器会根据闭包对变量的使用方式自动推断捕获方式，只有在需要强制获取所有权时才需要加上 `move`。

## 捕获的独占性

不可变借用和可变借用的规则同样适用于闭包捕获：

```Rust
fn main() {
    let mut list = vec![1, 2, 3];
    println!("Before defining closure: {:?}", list);

    // 只读闭包
    let only_borrows = || println!("From closure: {:?}", list);
    println!("Before calling closure: {:?}", list);
    only_borrows();

    // 到这里 only_borrows 已经不再使用，list 的不可变借用释放
    // 所以可以再创建一个可变借用闭包
    let mut borrows_mutably = || list.push(7);
    borrows_mutably();
    println!("After calling closure: {:?}", list);
}
```

# Fn、FnMut、FnOnce

Rust 闭包的捕获行为由三个 trait 来描述，所有闭包都至少实现了其中一个：

| Trait | 捕获方式 | 调用限制 |
|-------|---------|---------|
| `FnOnce` | 获取所有权 | 只能调用一次 |
| `FnMut` | 可变引用 | 可以多次调用，可修改捕获的变量 |
| `Fn` | 不可变引用 | 可以多次调用，不能修改捕获的变量 |

它们的继承关系为：`Fn` <: `FnMut` <: `FnOnce`。也就是说：

- 实现了 `Fn` 的闭包也自动实现了 `FnMut` 和 `FnOnce`
- 实现了 `FnMut` 的闭包也自动实现了 `FnOnce`

## FnOnce

```Rust
fn fn_once<F>(func: F)
where
    F: FnOnce(),
{
    func()
}

fn main() {
    let x = vec![1, 2, 3];
    // 消费了 x 的所有权
    let consume = || {
        drop(x);
    };
    fn_once(consume);
    // consume(); // 报错：consume 已经被消费
}
```

`FnOnce` 会在第一次调用时消费掉捕获的变量，因此只能调用一次。

## FnMut

```Rust
fn do_twice<F>(mut func: F)
where
    F: FnMut(),
{
    func();
    func();
}

fn main() {
    let mut list = vec![1, 2, 3];
    let mut borrows_mutably = || list.push(7);
    do_twice(borrows_mutably);
    println!("{:?}", list); // [1, 2, 3, 7, 7]
}
```

`FnMut` 可以多次调用，且允许修改捕获的变量。

## Fn

```Rust
fn do_twice<F>(func: F)
where
    F: Fn(),
{
    func();
    func();
}

fn main() {
    let list = vec![1, 2, 3];
    let only_borrows = || println!("{:?}", list);
    do_twice(only_borrows);
}
```

`Fn` 可以多次调用，但不能修改捕获的变量。

## 作为函数参数

在编写函数或方法时，如果需要接收一个闭包参数，应该根据需要选择合适的 trait bound：

- 只需要调用一次 → `FnOnce`
- 需要多次调用且可能修改环境 → `FnMut`
- 需要多次调用且不修改环境 → `Fn`

大多数情况下建议使用 `Fn`，如果需要修改环境再换成 `FnMut`，只有在明确知道只调用一次时才使用 `FnOnce`。

# 闭包作为返回值

闭包也可以作为函数的返回值。但由于闭包的大小在编译期不确定（每个闭包是唯一的匿名类型），不能直接返回闭包类型，需要使用 `impl Trait` 语法或者 `Box<dyn Trait>`：

```Rust
fn returns_closure() -> impl Fn(i32) -> i32 {
    |x| x + 1
}

fn main() {
    let f = returns_closure();
    println!("{}", f(1)); // 2
}
```

如果需要返回不同的闭包类型，则需要使用 trait 对象：

```Rust
fn returns_closure(flag: bool) -> Box<dyn Fn(i32) -> i32> {
    if flag {
        Box::new(|x| x + 1)
    } else {
        Box::new(|x| x * 2)
    }
}
```
