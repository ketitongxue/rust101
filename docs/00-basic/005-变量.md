Rust 中的变量默认是不可变的。

### 变量命名

Rust 的变量命名规范主要来自官方风格指南 **Rust Style Guide** 和社区约定。整体原则是：**snake_case + 语义清晰 + 避免缩写**。

1. 变量使用 `snake_case`
2. 可变变量命名没有特殊区别
3. 忽略变量使用 `_`
4. boolean 变量建议使用 `is_ / has_ / can_`
5. 避免缩写——Rust 社区偏向 **完整语义**。
6. shadowing 允许复用变量名

Rust 语言有一些**关键字**，和其他语言一样，这些关键字都是被保留给 Rust 语言使用的，因此，它们不能被用作变量或函数的名称。

Rust 官方把关键字分为三类：

- **Strict keywords（严格关键字）**：绝对不能作为变量名
- **Reserved keywords（保留关键字）**：为未来语法保留
- **Weak keywords（弱关键字）**：某些上下文中是关键字

1. **Strict keywords（严格关键字）**

|关键字|含义|作用示例|
|---|---|---|
|`as`|类型转换|`let x = 5 as f64;`|
|`break`|跳出循环|`break;`|
|`const`|定义常量|`const MAX: u32 = 100;`|
|`continue`|跳过本次循环|`continue;`|
|`crate`|当前 crate 根模块|`use crate::module;`|
|`else`|条件分支|`if x { } else { }`|
|`enum`|定义枚举类型|`enum Color { Red, Blue }`|
|`extern`|外部函数或 ABI|`extern "C" fn foo()`|
|`false`|布尔值 false|`let b = false;`|
|`fn`|定义函数|`fn add(a: i32, b: i32) -> i32 {}`|
|`for`|for 循环|`for i in 0..10 {}`|
|`if`|条件判断|`if x > 0 {}`|
|`impl`|实现 trait 或方法|`impl User {}`|
|`in`|迭代语法|`for x in vec {}`|
|`let`|定义变量|`let x = 5;`|
|`loop`|无限循环|`loop {}`|
|`match`|模式匹配|`match x {}`|
|`mod`|声明模块|`mod utils;`|
|`move`|move 闭包捕获|`move|
|`mut`|可变变量|`let mut x = 10;`|
|`pub`|公共可见性|`pub fn foo()`|
|`ref`|引用绑定|`let ref x = y;`|
|`return`|返回函数值|`return x;`|
|`self`|当前实例|`self.name`|
|`Self`|当前类型|`fn new() -> Self`|
|`static`|静态变量|`static COUNT: i32 = 0;`|
|`struct`|定义结构体|`struct User {}`|
|`super`|父模块|`super::module`|
|`trait`|定义 trait|`trait Display {}`|
|`true`|布尔值 true|`let b = true;`|
|`type`|类型别名|`type Id = u64;`|
|`unsafe`|不安全代码块|`unsafe {}`|
|`use`|引入模块|`use std::io;`|
|`where`|泛型约束|`where T: Display`|
|`while`|while 循环|`while x < 10 {}`|

2. **Reserved keywords（保留关键字）**

|关键字|含义 / 设计目的|说明|
|---|---|---|
|`abstract`|抽象类型 / 抽象类|计划用于类似 OOP 的抽象类型，但 Rust 目前用 `trait` 实现|
|`become`|尾调用优化|设计用于尾递归优化（tail call），目前未实现|
|`box`|堆分配语法|曾用于 `Box<T>` 的语法糖，现在暂未稳定|
|`do`|代码块表达式|曾设计为 `do` 表达式或 DSL 语法，目前未使用|
|`final`|最终类型或方法|类似 Java/C# 的 `final`，Rust 目前没有该概念|
|`macro`|宏定义关键字|Rust 实际使用 `macro_rules!`，未来可能简化为 `macro`|
|`override`|方法重写|设计用于 trait 或继承体系中的 override|
|`priv`|私有可见性|Rust 目前用 `pub` / 默认 private，因此该关键字未使用|
|`try`|错误处理|曾用于 try 表达式，现在 Rust 使用 `?` 运算符|
|`typeof`|获取类型|类似 C/C++ `typeof`，Rust 使用 `std::any::type_name` 等替代|
|`unsized`|未定长类型|用于描述 DST（dynamically sized types），目前内部机制|
|`virtual`|虚函数|Rust trait 已经实现多态，不需要该关键字|
|`yield`|生成器|未来 generator / coroutine 语法使用|

3. Weak keywords（弱关键字）

|关键字|含义|什么时候是关键字|什么时候可以当变量|
|---|---|---|---|
|`union`|定义联合体类型|`union MyUnion {}`|普通变量名|
|`dyn`|动态 trait 对象|`dyn Trait`|普通变量名|
|`'static`|静态生命周期|`&'static str`|不是变量名|
|`async`|异步函数/块|`async fn` / `async {}`|普通变量名|
|`await`|等待 future|`.await`|普通变量名|
|`try`|try block（实验特性）|`try {}`|变量名|
|`macro_rules`|定义宏|`macro_rules!`|变量名|

### 变量绑定

在其它语言中，我们用 `var a = "hello world"` 的方式给 `a` 赋值，也就是把等式右边的 `"hello world"` 字符串赋值给变量 `a` ，而在 Rust 中，我们这样写： `let a = "hello world"` ，同时给这个过程起了另一个名字：**变量绑定**。

为何不用赋值而用绑定呢（其实你也可以称之为赋值，但是绑定的含义更清晰准确）？这里就涉及 Rust 最核心的原则——**所有权**，简单来讲，任何内存对象都是有主人的，而且一般情况下完全属于它的主人，绑定就是把这个对象绑定给一个变量，让这个变量成为它的主人（聪明的读者应该能猜到，在这种情况下，该对象之前的主人就会丧失对该对象的所有权），像极了我们的现实世界，不是吗？

### 使用下划线开头忽略未使用的变量

如果你创建了一个变量却不在任何地方使用它，Rust 通常会给你一个警告，因为这可能会是个 BUG。但是有时创建一个不会被使用的变量是有用的，比如你正在设计原型或刚刚开始一个项目。这时**你希望告诉 Rust 不要警告未使用的变量，为此可以用下划线作为变量名的开头**：

```Rust
fn main() {
    let _x = 5;
    let y = 10;
}
```

使用 `cargo run` 运行下试试:

值得注意的是，这里编译器还很善意的给出了提示（Rust 的编译器非常强大，这里的提示只是小意思）：将 `y` 修改 `_y` 即可。

### 变量解构

`let` 表达式不仅仅用于变量的绑定，还能进行复杂变量的解构：从一个相对复杂的变量中，匹配出该变量的一部分内容：

```Rust
fn main() {
    let (a, mut b): (bool,bool) = (true, false);
    // a = true,不可变; b = false，可变
    println!("a = {:?}, b = {:?}", a, b);

    b = true;
    assert_eq!(a, b);
}
```

解构式赋值——模式赋值（Destructuring assignments）

我们可以在赋值语句的左式中使用元组、切片和结构体。

```Rust
struct Struct {
    e: i32
}

fn main() {
    let (a, b, c, d, e);

    (a, b) = (1, 2);
    // _ 代表匹配一个值，但是我们不关心具体的值是什么，因此没有使用一个变量名而是使用了 _
    [c, .., d, _] = [1, 2, 3, 4, 5];
    Struct { e, .. } = Struct { e: 5 };

    assert_eq!([1, 2, 1, 4, 5], [a, b, c, d, e]);
}
```

这种使用方式跟之前的 `let` 保持了一致性，但是 `let` 会重新绑定，而这里仅仅是对之前绑定的变量进行再赋值。

### 变量遮蔽（shadowing）

Rust 允许声明相同的变量名，在后面声明的变量会遮蔽掉前面声明的，如下所示：

```Rust
fn main() {
    let x = 5;
    // 在main函数的作用域内对之前的x进行遮蔽
    let x = x + 1;

    {
        // 在当前的花括号作用域内，对之前的x进行遮蔽
        let x = x * 2;
        println!("The value of x in the inner scope is: {}", x);
    }

    println!("The value of x is: {}", x);
}
```

在 Rust 中，当你看到 `let` 关键字再次出现时，就意味着一个**新变量**诞生了。虽然看起来你是在修改同一个变量，但实际上，编译器在背后**创建了一个全新的变量**，只是它碰巧和之前的变量同名。

`mut` 声明的变量，可以修改同一个内存地址上的值，并不会发生内存对象的再分配，性能要更好。

这个程序首先将数值 `5` 绑定到 `x`，然后通过重复使用 `let x =` 来遮蔽之前的 `x`，并取原来的值加上 `1`，所以 `x` 的值变成了 `6`。第三个 `let` 语句同样遮蔽前面的 `x`，取之前的值并乘上 `2`，得到的 `x` 最终值为 `12`。当运行此程序，将输出以下内容：

```Rust
/Users/keti/.cargo/bin/cargo run --color=always --package variables --bin variables --profile dev
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 0.00s
     Running `target/debug/variables`
The value of x in the inner scope is: 12
The value of x is: 6
```
